# guanfu · Tier 2 — 决策框架 + 行为护栏

> 前置:已读 `tier1.md`。本文档聚焦**怎么把 tier1 的原始数据翻译成结构化建议**,以及如何给建议时不造成负担。

## 1. 读盘工作流(9 步,按决策影响顺序)

不要从前到后读盘,按决策影响顺序:

1. **周期位置** — `days_since_halving` + `sma_200w_dev` + `phase`(accumulation / markup / distribution)
2. **估值一致性** — `ahr999_compressed` + `mayer_multiple` + `sma_200w_dev` + `pi_cycle_top_ratio` 4 项交叉,**4 项都指低估才叫低估**
3. **网络健康** — `hash_ribbons` + `difficulty_change_pct` + `mempool_mb`
4. **杠杆健康** — `funding_rate_pct` + `oi_to_mc`(避免高拥挤区追涨)
5. **宏观环境** — m2 / real yield / DXY / spx_correlation
6. **流入** — ETF 30d / stablecoin / ETH_BTC
7. **技术指标** — RSI / MACD / MA / BB(短期方向确认)
8. **跨资产** — BTC / Gold / QQQ / SPY 相关性 / 相对强弱
9. **图表输出**(可选)— 价格 + 均线 + 估值带 / 8 域雷达 / V/M/P 仪表盘

## 2. 域级方向规则

8 个域,每域统计看涨 / 看跌指标数,得出域级方向(+1 / 0 / -1)。

| 域 | 看涨(+1) | 看跌(-1) |
|---|---|---|
| Cycle | mayer<1.0 或 sma_200w_dev<0 | pi_cycle≥0.85 或 sma_200w_dev>1.5 |
| Valuation | ahr999_compressed<0.846 或 MVRV Z<0 | ahr999_compressed>3.344 或 MVRV Z>5 |
| Network | hash_ribbons 上行 或 difficulty>+5% | hash_ribbons 下行 + difficulty<-5% |
| Positioning | ≥2 确认:funding<0 / OI/MC<0.015 / F&G<25 | ≥2 确认:funding>0.05 / OI/MC>0.035 / F&G>80 |
| Macro | m2>5 / real_yield<1 / DXY60d<-1 中 ≥2 | m2<0 / real_yield>2.5 / DXY60d>+1 中 ≥2 |
| Flow | ETF30d>$1B 主信号 | ETF30d<-$1B 主信号 |
| Technical | MACD>0 / EMA12>EMA26 / MA50>MA200 中 ≥2 | 对偶 |
| CrossAsset | BTC/SPY corr<0.3 + BTC 90d 跑赢 Gold | BTC/SPY corr>0.7 + BTC 90d 跑输 Gold |

**净方向** ∈ [-8, +8]:
- ≥+5 → 强积累倾向
- +3~+4 → 偏积累
- +1~+2 → 持有观察
- 0 → 等待
- -1~-2 → 防守倾向
- -3~-4 → 高防守
- ≤-5 → 分配/避险

## 4. 行为护栏(必出 ≥3 条)

投资失败 80% 不是分析错了,是行为错误。每次读盘**必须**附带至少 3 条护栏,从菜单按场景选:

### 冷静期检查
若同一资产 < 4h 内多次被问 → "行情波动驱动反复读盘是过度反应风险,建议 15 分钟后再看"

### 对立论证强制
- bull 结论 → 必须附 "最强反方论点是 X"
- bear 结论 → 必须附 "最强正方论点是 X"

### 锚定偏差检查(决策型问题必出)
"**忽略你当前的成本基础**,如果你今天有和持仓等值的现金,你还会买吗?"

### 期限一致性
若用户(portfolio.yaml)声明 5y 期限但在问 30d forecast → "30d 对 5y 期限的你是噪声,请优先读 180d+ horizon"

### 确认偏差检查
"读盘前,你**希望**答案是什么?读盘后对比是否相符,不符时请认真看反证"

### FOMO 检查
若价格 30d 涨 > 20% 且用户不在持仓 → "追涨是统计上的负期望行为,请先读触发条件,未满足不追"

### 恐慌检查
若价格 30d 跌 > 20% 且用户满仓 → "恐慌卖是锁定亏损,请先读触发条件"

## 5. 给建议的措辞规范(最关键)

过于肯定的话术是使用者的负担。以下是强制规范:

| 禁用 | 替代 |
|---|---|
| "BTC 会涨 / 会跌" | "基于 A/B/C 三条证据,倾向 X,概率约 Y%" |
| "现在是底部" | "基于 [指标],符合历史低估区特征,概率约 X%;反证为 [Y]" |
| "一定要买 / 必须卖" | "若你是 [画像] + 确认 [条件],证据支持 [动作倾向]" |
| "目标价 $120k" | "合理估值区间 $[L]-$[H],失效条件 [Z]" |
| "止损设 $70k" | "若 [指标] 跌破 [阈值] + [另一指标] 同向,结论失效,需重新读盘" |
| "仓位建议 30%" | 不建议仓位百分比;组合参数由用户 portfolio.yaml 或明确声明提供 |

## 6. 必设失效条件(硬规则)

**每条结论**必须附失效条件,格式:"若 [指标 / 域] 变化到 [阈值 / 状态],本结论失效,应重新读盘"

示例:
- "若 mayer_multiple 跌破 0.6 且 hash_ribbons 下行,本低估结论失效(机构 / 矿工双弱 = 深度压力)"
- "若 real_yield_10y 突破 2.5% 连续 1 个月,本积累倾向失效(宏观估值锚抬升)"
- "若 ETF 连续 4 周净流出且 stablecoin 收缩,本流入支撑失效"

## 7. 反模式速查

- ❌ 看 1 个指标做决策
- ❌ 4 个估值指标挑利己的用
- ❌ 忽略 stale 警告
- ❌ 黑天鹅 / 监管事件还看 guanfu(超出指标范围)
- ❌ 用 guanfu 决策 altcoin(仅覆盖 BTC + ETH/BTC 比率)
- ❌ 输出倾向但不列反证
- ❌ 用绝对语气
- ❌ 不设失效条件
- ❌ hard_blocked horizon 还输出数值

## 8. 跨资产比较(`market` 子命令的 AI 解读)

"现在有 1 块闲钱,BTC / QQQ / Gold 哪个证据更充分?"
- **不打分**,不排序
- 列三资产各自的 "支持证据 TOP2" + "反证 TOP1"
- 附 "哪个的 reliability 更高 / 哪个 source_health 更干净"
- 让用户自己选
