## 一、同指标精度对比：AHR999

AHR999 是市场上最常用的 BTC 估值指标。以下是传统公式和 guanfu 改进版的差异。

### 公式差异

| | 传统 AHR999 (ahr999-free 等) | guanfu 改进版 |
|---|---------------------------|-------------|
| 定投成本 | `SMA(price, 200)` — 算术均值 | `200 / Σ(1/price_i)` — **调和均值** |
| 公平值拟合 | 固定系数：`5.84×log₁₀(days) - 17.01`（2019 年标定） | **Huber IRLS 动态加权回归**，8 年窗口，半衰期可配 |
| 评分阈值 | 固定：0.45 / 0.8 / 1.0 / 1.2 / 2.0 | **动态分位数**：q10/q35/q55/q75/q90 |
| 抗 outlier | 无（LS 对极端值敏感） | ✅ Huber 重加权（降 2021 顶/LUNA 双爆权重） |

### 当前值对比

```
BTC 价格: $78,210
200d 算术 MA: ~$83,871（传统 DCA 成本）
200d 调和 DCA: ~$81,420（guanfu DCA 成本，修正约 +3%）

传统 AHR999 = (78210/83871) × (78210/固定exp_fit)
guanfu AHR   = (78210/81420) × (78210/动态fair_value) ← DCA更低，fair_value更新
```

**关键差异**：传统公式的固定系数基于 2019 年之前的数据标定，8 年未更新。2020-2021 牛市、2022 LUNA/FTX、2024 ETF——这些结构性事件全部不在系数里。guanfu 的动态拟合每运行一次都用最新 8 年数据重算。

## 三、实际分析深度对比

以 2026-05-02 盘面为例，同一时刻各工具能给出的信息：

### ahr999-free
```
AHR999 = ~0.71
→ 定投区（0.45-1.2）
```

**问题**：只有一个数。不知道为什么是 0.71，不知道这个数在历史上什么位置（q 分位），不知道其他维度在说什么。

### btc-ai-advisor
```
AHR999 = 0.71 (定投区)
F&G = 39 (恐慌)
→ AI 建议：建议观望/定投
```

**问题**：5 个指标，2 个域。没有网络数据（矿工在干什么？），没有宏观（M2 在扩张吗？），没有资金流（ETF 还在买吗？），没有跨资产对比（BTC 相对 QQQ 是强是弱？）。

### BRK
```
MVRV = 1.44
NUPL = 0.31
哈希率 = 1067 EH/s
... (数百个链上指标)
```

**问题**：链上数据极深，但缺乏交易数据（funding/OI/ETF flow）、宏观数据（DXY/M2/real_yield）、跨资产对比。且需要跑全节点（~700GB 磁盘）。

### guanfu
```
8 域 42 指标，12/12 交叉验证通过。

核心判断：10/10 顶部指标不满足 → 不是顶
关键矛盾：mayer 0.93(q20) 偏低估，但 halving 742d 传统模板为晚期
杠杆状态：funding -0.005% + OI/MC 0.005 → 杠杆已完全清洗
网络信号：hash_ribbons 下行 → 历史命中率 100% 的底部前奏
跨资产：BTC/QQQ corr ~0，QQQ 跌 21% 时 BTC 涨 1.6% → 独立行情确认
技术面：EMA 多头 + MA 死叉 = 短多中空，波动率 1.92% 极低 → 1-2 周内变盘

信号计分：+4/-8~+8 → 🔵 ACCUMULATE
历史回测：最相似 4 时期平均 +68.9% 90 日收益
```
